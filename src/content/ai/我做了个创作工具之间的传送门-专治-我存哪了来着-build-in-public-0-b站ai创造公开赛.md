**来源**：\[转录/我做了个创作工具之间的传送门，专治我存哪了来着【B站AI创造公开赛】\]\]( / B 站 / UP：Cora 团队（Creaibo）

**核心结论**：Cora 是一款浏览器插件，专门解决 AI 创作者在多个工具间反复拖拽提示词/分镜/图片/视频资产的痛点，背后论断是「创作的本质就是上下文对齐」；后续版本会引入 Creable Agent 做资产自动分类、规划与管理。

**核心定位**

| 维度 | 内容 |
|----|----|
| 形态 | 浏览器插件（Chrome / 金融商店），Beta 已开放免费下载 |
| 解决场景 | Notion / 飞书 / Lowart / Type Now / 剪映 等工具间上下文流转 |
| 可流转资产 | 提示词 / Skill / 人物图片 / 画布节点 / 视频资产 |
| 工作流示例 | Notion 故事 → Creaibo Agent 共创 → 飞书文档协作 → Lowart 生图 → Type Now 视频 → 剪映后期 |
| 后续路线 | 本地存储 + Creable Agent 自动分类规划；与本地剪辑软件打通做素材预处理 |
| 开发栈 | Codex + Claude Code |
| 关联比赛 | B 站 AI 创造公开赛 Build in Public 系列第 0 期 |

**对比判断**

- vs Make/n8n 等工作流自动化工具：Cora 走浏览器插件 + 拖拽，比工作流更轻、不需搭流程图
- vs 飞书多维表/Notion 库内资产：Cora 跨工具而非单工具内，定位是"工具间传送门"
- vs 个人云盘手动归集：Cora 强调实时拖拽 + Agent 后续接管归类

**写入边界**

- 视频是产品演示 + 创业分享，Cora 内测名额、Cora vs 自建类似脚本的真实效率差等关键信息未在视频中给出
- "上下文对齐"是 Cora 自家论断，缺少第三方对比或效率数据
- 团队仍在持续招聘，团队规模、付费计划、隐私策略暂未公开
- 短链/官网 creaibo.com 长期可用性需观察
